Extensions go in the extensions/ folder next to the exe, one subfolder per extension with a main.lua inside.
--------------------------
  extensions/
    my-engine/
      main.lua
--------------------------
The standard Lua library is available. dofile(), loadfile() and require() resolve relative paths
against your extension's own folder first, then the app folder, so an extension can split itself
across several files in its own folder and pull them in by name.
--------------------------
Every extension needs an Extension() function, OpenSpeaker calls it on load to get metadata.
id and name are required, description is optional.
--------------------------
function Extension()
	return {
		id          = "my-engine",
		name        = "My Engine",
		description = "does stuff",
	}
end
--------------------------
To register a speech engine (shows up in Speech Engines > Add).
auth fields only support type "string" for now. label is optional, it defaults to a prettified key.
A positional form { "api_key", "string" } also works.
--------------------------
RegisterSpeechEngine({
	id   = "my-engine",
	name = "My Engine",
	auth = {
		{ key = "api_key",    label = "API Key",    type = "string" },
		{ key = "api_secret", label = "API Secret", type = "string" },
	}
})
--------------------------
Everywhere OpenSpeaker stores an engine id it prefixes it with "ext:", so the engine above is
"ext:my-engine" in voice history, aliases and migration hooks. The engine_id passed to your
GetVoices/GenerateSpeech callbacks, and the one you pass to GetAuth, is the raw unprefixed id.
--------------------------
To add a settings panel under the Extensions tab use RegisterSettings().
types: text, number, checkbox, dropdown (dropdown requires options), keybind
label is optional and defaults to a prettified key.
--------------------------
RegisterSettings({
	{ key = "quality", label = "Quality", type = "dropdown", options = {"standard", "premium"}, default = "standard" },
	{ key = "speed",   label = "Speed",   type = "number",   default = 1.0 },
	{ key = "on",      label = "On",      type = "checkbox", default = true },
	{ key = "prefix",  label = "Prefix",  type = "text",     default = "" },
	{ key = "say_hi",  label = "Say Hi",  type = "keybind",  default = "F8" },
})
--------------------------
If you were using another engine as a workaround in old SpeakerBot, RegisterMigrationHook() tells
the importer to remap that engine's voices and config to yours during import.
remap_voices remaps voice history and alias voices. remap_config translates the engine credentials,
with fields mapping your field names to the SpeakerBot source field names.
--------------------------
RegisterMigrationHook({
	remap_voices = { uberduck = "ext:my-engine" },
	remap_config = {
		uberduck = {
			engine_id = "ext:my-engine",
			fields = { api_key = "clientId", api_secret = "secretKey" }
		}
	}
})
--------------------------
If you registered a speech engine you need to define this function to fill out the voice list.
voice fields: id (required), name (required), locale (optional), gender (optional)
Return {} if auth is missing or wrong. Anything that is not a non-empty array table is treated
as "no voices" and logged.
--------------------------
function GetVoices(engine_id)
	return {
		{ id = "v1", name = "Alice", locale = "en-US", gender = "Female" },
	}
end
--------------------------
This callback will be called by OpenSpeaker when it needs to generate speech.
The alias argument is reserved and is currently always nil.

Return one of:
	"https://..."                            url, openspeaker fetches it, format guessed from the url
	{ url = "https://...", format = "mp3" }  url with explicit format
	{ format = "mp3", data = "<base64>" }    raw audio bytes as base64
	{ async_job = job_id, format = "mp3" }   job id from http.get_bytes_async / http.post_bytes_async
	nil                                      failed, skip this request

Only "wav" and "mp3" are decodable. Anything that is not "wav" is decoded as mp3.
Returning an async_job id lets the download run outside the Lua lock, so other extension callbacks
are not blocked while the audio is in flight. The id is consumed by the GenerateSpeech call that
returns it, so start the job and return its id from the same call.
--------------------------
function GenerateSpeech(engine_id, voice_id, text, alias)
	local job = http.post_bytes_async("https://api.example.com/tts", json.stringify({ text = text }))
	return { async_job = job, format = "mp3" }
end
--------------------------
Optionally define OnMessage to intercept chat messages before they get spoken.
user fields: id, username, display_name, nickname, is_subscriber, is_mod, is_vip,
			is_broadcaster, is_regular, is_ignored, is_forced

Return a string to replace the message, or "" to block it entirely.
Returning nil (or erroring) passes the original message through unchanged, it does NOT block.
If several extensions define OnMessage they are chained in load order, each one receiving the
previous one's output.
--------------------------
function OnMessage(user, message)
	if user.is_mod then return message end
	return message:gsub("badword", "***")
end
--------------------------
Optionally define OnUpdate, called about 60 times per second while OpenSpeaker runs.
It only runs if your extension defines OnUpdate or registers a keybind setting.
If your OnUpdate is still running when the next tick comes around, that tick is skipped instead of
being queued up. A slow OnUpdate just runs less often, it never stacks up or runs twice at once.
--------------------------
function OnUpdate()
	if keybind.pressed("say_hi") then
		chat.send("Hi chat!")
	end
end
--------------------------
Keybinds
--------------------------
Keys are read from a global hook, so binds fire while OpenSpeaker is unfocused. Input going to a
window running as administrator is invisible unless OpenSpeaker is also running as administrator.

The argument to keybind.* is either the key of a "keybind" setting (resolving to whatever the user
bound in the UI) or a literal spec like "F8" or "Ctrl+Shift+K".

Combos are joined with "+" and every key must be held at once. Any keys work, not just modifiers,
so "K+U+Y" is a valid bind. "Ctrl", "Shift", "Alt" and "Win" match either side, use "LCtrl",
"RShift", "LAlt" etc. to require a specific one. Bind the plus key itself as "Plus" or "++".

keybind.held(key)      true while every key in the combo is down
keybind.pressed(key)   true on the tick the combo became fully held
keybind.released(key)  true on the tick the combo stopped being fully held

pressed/released are edge triggered against a recorded key event log, not sampled state, so a tap
shorter than one tick still reports both on the same tick and is never dropped. An unrecognised or
empty spec is always false.
--------------------------
Chat
--------------------------
chat.send(msg)  sends a message to the connected Twitch chat, no-op until chat is connected
--------------------------
Settings and auth
--------------------------
GetAuth(engine_id)   returns a table of the auth values the user entered for that engine
GetSetting(key)      returns a single setting value
GetSettings()        returns all settings as a table
GetVoiceSettings()   reserved, currently always returns an empty table
log(msg)             writes to the debug log

Setting values come back typed: checkbox fields are booleans, number fields are numbers, and
everything else (text, dropdown, keybind) is a string. Unset fields fall back to their default.
--------------------------
http
--------------------------
http.get(url, headers?)                             -> { ok, status, body }
http.post(url, body, contentType?, headers?)        -> { ok, status, body }
http.get_bytes(url, headers?)                       -> { ok, status, data }   data is base64
http.post_bytes(url, body, contentType?, headers?)  -> { ok, status, data }   data is base64
http.get_bytes_async(url, headers?)                 -> job_id
http.post_bytes_async(url, body, contentType?, headers?) -> job_id

contentType defaults to "application/json" if omitted.
headers is a plain table like { ["Authorization"] = "Bearer xyz" }

On a network failure the blocking calls return ok = false and status = 0. The body/data field holds
the error message for get/post and an empty string for the byte variants.

The _async variants return immediately with a job id string. They do not block the Lua state, and
the only way to consume one is to return it as async_job from GenerateSpeech. A job id that is
never returned keeps its download running and is cleaned up when the extension unloads.
--------------------------
json / base64
--------------------------
json.parse(str)      parses json string into a lua value, nil if the string is not valid json
json.stringify(val)  serializes a lua value to a json string, "null" if it cannot be serialized

base64.encode(str)   utf8 string -> base64
base64.decode(str)   base64 -> utf8 string, "" if the input is not valid base64

json.stringify treats a table with a 1..n array part as a json array and any other table as a json
object. An empty table serializes as an object.
